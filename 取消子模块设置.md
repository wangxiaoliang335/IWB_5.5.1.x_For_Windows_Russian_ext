# 取消 IWB 子模块设置并重新提交

## 目标

将 IWB 从子模块转换为普通目录，然后重新提交。

## 操作步骤

### 步骤 1: 从 Git 中移除子模块

```bash
# 1. 从 .git/modules 中移除子模块（如果存在）
rm -rf .git/modules/IWB

# 2. 从 .git/config 中移除子模块配置
git config -f .git/config --remove-section submodule.IWB 2>/dev/null || true

# 3. 从 .gitmodules 中移除（如果存在）
git rm --cached .gitmodules 2>/dev/null || true
rm -f .gitmodules

# 4. 从索引中移除子模块
git rm --cached IWB
```

### 步骤 2: 移除 IWB 的 .git 目录

```bash
# 移除 IWB 目录中的 .git（保留所有文件）
rm -rf IWB/.git
```

### 步骤 3: 重新添加 IWB 作为普通目录

```bash
# 添加 IWB 目录
git add IWB

# 提交
git commit -m "Convert IWB from submodule to regular directory"

# 推送
git push github-ext master
```

## 完整命令序列

```bash
# 1. 移除子模块配置
rm -rf .git/modules/IWB
git config -f .git/config --remove-section submodule.IWB 2>/dev/null || true
rm -f .gitmodules

# 2. 从索引中移除子模块
git rm --cached IWB

# 3. 移除 IWB 的 .git 目录
rm -rf IWB/.git

# 4. 重新添加为普通目录
git add IWB

# 5. 提交
git commit -m "Convert IWB from submodule to regular directory"

# 6. 推送
git push github-ext master
```

## 注意事项

- 这不会删除 IWB 目录中的任何文件
- 只是将 IWB 从子模块转换为普通目录
- 确保 IWB/.git/ 在 .gitignore 中（已经添加了）
