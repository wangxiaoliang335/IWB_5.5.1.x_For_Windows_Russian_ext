# GitHub 仓库设置操作指南

## 已完成的工作

我已经为您创建了以下文件：

1. **`.gitignore`** - 自动排除大文件和构建产物
2. **`setup_github_repo.bat`** - Windows批处理脚本，用于设置远程仓库
3. **`setup_github_repo.py`** - Python脚本，用于设置远程仓库
4. **`check_large_files.bat`** - 检查大于50M的文件
5. **`README_GIT_SETUP.md`** - 详细的设置说明文档

## 手动操作步骤

由于自动执行可能遇到环境问题，您可以按照以下步骤手动操作：

### 步骤 1: 初始化 Git 仓库（如果还没有）

打开命令提示符（CMD）或 PowerShell，进入项目目录，然后运行：

```bash
git init
```

### 步骤 2: 添加新的远程仓库

运行以下命令添加您的 GitHub 仓库：

```bash
git remote add github git@github.com:wangxiaoliang335/IWB_5.5.1.x_For_Windows_Russian.git
```

如果 `github` 这个远程名称已存在，可以使用：

```bash
git remote set-url github git@github.com:wangxiaoliang335/IWB_5.5.1.x_For_Windows_Russian.git
```

### 步骤 3: 验证远程仓库配置

查看所有远程仓库：

```bash
git remote -v
```

您应该看到类似以下输出：

```
github  git@github.com:wangxiaoliang335/IWB_5.5.1.x_For_Windows_Russian.git (fetch)
github  git@github.com:wangxiaoliang335/IWB_5.5.1.x_For_Windows_Russian.git (push)
```

### 步骤 4: 配置 .gitignore

`.gitignore` 文件已经创建好了，它会自动排除：
- 所有构建目录（bin/, Debug/, Release/ 等）
- 中间文件（*.obj, *.pdb 等）
- 压缩文件（*.zip 等）
- 可执行文件（*.exe, *.dll 等）

如果还有大于 50M 的文件需要排除，可以运行：

```bash
check_large_files.bat
```

然后将文件路径添加到 `.gitignore` 文件中。

### 步骤 5: 添加文件并提交

```bash
git add .
git commit -m "Initial commit: IWB 5.5.1.x Russian version"
```

### 步骤 6: 推送到 GitHub

```bash
git push github main
```

或者如果您的默认分支是 `master`：

```bash
git push github master
```

## 关于文件大小限制

`.gitignore` 已经配置为排除常见的大文件。GitHub 对单个文件有 100MB 的限制，超过 50MB 的文件建议：

1. 添加到 `.gitignore` 中排除
2. 或使用 Git LFS（Large File Storage）来管理大文件

## 如果遇到 SSH 连接问题

如果您还没有配置 SSH 密钥，可以使用 HTTPS 地址：

```bash
git remote set-url github https://github.com/wangxiaoliang335/IWB_5.5.1.x_For_Windows_Russian.git
```

使用 HTTPS 时，推送时需要输入 GitHub 用户名和访问令牌（Personal Access Token）。

## 常用命令

- 查看远程仓库：`git remote -v`
- 查看状态：`git status`
- 查看哪些文件会被提交：`git status`
- 查看 .gitignore 是否生效：`git status --ignored`

## 注意事项

1. 确保 `.gitignore` 文件在项目根目录
2. 如果文件已经在 Git 跟踪中，需要先从 Git 中移除：
   ```bash
   git rm --cached <文件名>
   ```
3. 大文件（如 .zip 文件）已经在 .gitignore 中，不会被提交

## 需要帮助？

如果遇到任何问题，请检查：
1. Git 是否已正确安装：`git --version`
2. SSH 密钥是否已配置（如果使用 SSH 地址）
3. GitHub 仓库是否存在且有推送权限
