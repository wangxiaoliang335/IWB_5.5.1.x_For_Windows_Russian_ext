# 立即取消 IWB 子模块设置

## 快速操作步骤

### 方法 1: 使用批处理脚本（推荐）

直接运行：
```bash
取消IWB子模块.bat
```

然后推送：
```bash
git push github-ext master
```

### 方法 2: 手动执行命令

按顺序执行以下命令：

```bash
# 1. 移除子模块配置
rm -rf .git/modules/IWB
git config -f .git/config --remove-section submodule.IWB 2>/dev/null || true
rm -f .gitmodules

# 2. 从索引中移除子模块
git rm --cached IWB

# 3. 移除 IWB 的 .git 目录（保留所有文件）
rm -rf IWB/.git

# 4. 重新添加 IWB 作为普通目录
git add IWB

# 5. 提交
git commit -m "Convert IWB from submodule to regular directory"

# 6. 推送
git push github-ext master
```

## 完整命令序列（Windows Git Bash）

```bash
# 步骤 1: 移除子模块配置

rm -f .gitmodules

# 步骤 2: 从索引中移除

# 步骤 3: 移除 IWB 的 .git 目录
rm -rf IWB/.git

# 步骤 4: 重新添加
git add IWB

# 步骤 5: 提交
git commit -m "Convert IWB from submodule to regular directory"

# 步骤 6: 推送
git push github-ext master
```

## 说明

- ✅ 不会删除 IWB 目录中的任何源代码文件
- ✅ 只是将 IWB 从子模块转换为普通目录
- ✅ 所有 IWB 的内容都会被正常提交
- ✅ IWB/.git/ 已经在 .gitignore 中，不会再次创建

## 验证

提交后，可以验证：

```bash
# 查看 IWB 是否作为普通目录被跟踪
git ls-files | grep "^IWB/"

# 查看状态（应该没有子模块相关的提示）
git status
```
