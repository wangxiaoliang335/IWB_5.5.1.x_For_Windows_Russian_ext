# 添加新的 GitHub 远程仓库

## 目标

添加新的远程仓库：`git@github.com:wangxiaoliang335/IWB_5.5.1.x_For_Windows_Russian_ext.git`

并推送代码（只提交小于50M的文件）

## 操作步骤

### 步骤 1: 添加新的远程仓库

```bash
# 添加新的远程仓库（使用不同的名称，比如 github-ext）
git remote add github-ext git@github.com:wangxiaoliang335/IWB_5.5.1.x_For_Windows_Russian_ext.git

# 或者如果想替换原来的 github 远程
git remote set-url github git@github.com:wangxiaoliang335/IWB_5.5.1.x_For_Windows_Russian_ext.git
```

### 步骤 2: 查看所有远程仓库

```bash
git remote -v
```

### 步骤 3: 确保 .gitignore 已配置

检查 `.gitignore` 是否已经配置了排除大于50M的文件：
- `*.zip` 等压缩文件
- 构建产物目录
- 其他大文件

### 步骤 4: 检查大文件

```bash
# 运行检查脚本
check_large_files.bat
```

### 步骤 5: 提交更改（如果需要）

```bash
# 查看状态
git status

# 添加文件
git add .

# 提交
git commit -m "Update: prepare for new repository"
```

### 步骤 6: 推送到新仓库

```bash
# 如果使用新的远程名称 github-ext
git push github-ext master

# 或者如果替换了 github 远程
git push github master
```

## 完整命令序列

```bash
# 1. 添加新远程仓库
git remote add github-ext git@github.com:wangxiaoliang335/IWB_5.5.1.x_For_Windows_Russian_ext.git

# 2. 验证
git remote -v

# 3. 检查大文件
check_large_files.bat

# 4. 查看状态
git status

# 5. 添加并提交（如果需要）
git add .
git commit -m "Prepare for new repository: IWB_5.5.1.x_For_Windows_Russian_ext"

# 6. 推送到新仓库
git push github-ext master
```

## 关于 SSH vs HTTPS

如果遇到 SSH 连接问题，可以改用 HTTPS：

```bash
# 改为 HTTPS 地址
git remote add github-ext https://github.com/wangxiaoliang335/IWB_5.5.1.x_For_Windows_Russian_ext.git
```
