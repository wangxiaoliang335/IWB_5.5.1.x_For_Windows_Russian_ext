# 移除大文件并重新推送

## 问题

GitHub 检测到文件 `.vs/IWB_VS2015/v16/Browse.VC.db` 超过 100MB 限制（192.37 MB）

## 解决方案

执行以下步骤：

### 步骤 1: 从 Git 索引中移除大文件

```bash
# 从 Git 索引中移除 .vs 目录（但保留本地文件）
git rm -r --cached .vs/

# 或者只移除特定的大文件
git rm --cached .vs/IWB_VS2015/v16/Browse.VC.db
```

### 步骤 2: 确保 .gitignore 已更新

我已经将 `.vs/` 目录添加到 `.gitignore` 文件中。

### 步骤 3: 重新提交

```bash
# 添加更新的 .gitignore
git add .gitignore

# 提交更改（移除大文件）
git commit -m "Remove large files: .vs directory and Browse.VC.db"
```

### 步骤 4: 推送到 GitHub

```bash
git push github-ext master
```

## 完整命令序列

```bash
# 1. 移除 .vs 目录从 Git 跟踪
git rm -r --cached .vs/

# 2. 添加更新的 .gitignore
git add .gitignore

# 3. 提交更改
git commit -m "Remove large files: exclude .vs directory (Visual Studio files)"

# 4. 推送到 GitHub
git push github-ext master
```

## 验证

推送后，执行以下命令验证大文件已被排除：

```bash
# 检查 .vs 目录是否还在 Git 跟踪中（应该没有输出）
git ls-files | grep "\.vs"
```

## 注意

- `.vs/` 目录是 Visual Studio 的用户特定文件，不应该提交到 Git
- 移除后，本地文件仍然存在，只是不再被 Git 跟踪
- 其他开发者克隆仓库后，Visual Studio 会自动重新生成这些文件
